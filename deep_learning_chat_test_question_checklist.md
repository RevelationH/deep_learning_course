## 18. 这份清单重点要捕捉的问题

- 上下文是否会在非追问轮次错误继承。
- 多概念问题是否只回答了一半。
- `代码 + 解释` 请求是否只输出代码块而缺少说明。
- 只要代码的请求是否仍然夹带长篇概念介绍。
- out-of-scope 问题是否被错误回答。
- reference 类问题是否把文件名和页码直接写进正文，而不是交给引用区展示。
- 引用区是否和正文内容一致。
- 重复提问时回答是否能保持事实一致但措辞不完全僵死。
- 会话记忆是否会被问候语或越界问题污染。
- 回答是否仍然带有明显的系统/检索/演示语气。
